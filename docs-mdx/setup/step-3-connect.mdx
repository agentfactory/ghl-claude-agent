# Step 3: Connect Claude to GHL

Now you connect Claude to your GHL account using the PIT token and Location ID from the previous steps. Pick the option that matches what you use.

## Option A: Claude Desktop (Recommended for Most Users)

Claude Desktop is the standalone app. This is the easiest path.

### Open the Config File

Open PowerShell and run this command:

```powershell
notepad "$env:APPDATA\Claude\claude_desktop_config.json"
```

If Notepad says the file does not exist, click **Yes** to create it.

**What you should see:** Notepad opens with either an empty file or an existing configuration.

### Add the GHL Connection

If the file is empty, paste this entire block. Replace `YOUR_PIT_TOKEN_HERE` with your actual PIT token from Step 1.

```json
{
  "mcpServers": {
    "ghl": {
      "url": "https://services.leadconnectorhq.com/mcp/",
      "headers": {
        "Authorization": "Bearer YOUR_PIT_TOKEN_HERE",
        "locationId": "YOUR_LOCATION_ID_HERE"
      }
    }
  }
}
```

Replace `YOUR_PIT_TOKEN_HERE` with your actual PIT token from Step 1. Replace `YOUR_LOCATION_ID_HERE` with the Location ID from Step 2.

If the file already has content, add the `ghl` entry inside the existing `mcpServers` section.

### Save and Restart

1. Save the file in Notepad (Ctrl+S)
2. Close Claude Desktop completely (check your system tray)
3. Reopen Claude Desktop

**What you should see:** A small hammer icon appears near the chat input area. This means Claude recognizes the MCP connection.

[SCREENSHOT: Claude Desktop chat input showing the hammer/tools icon]

<Warning>
  Make sure the JSON is valid. A missing comma or bracket will prevent it from loading. If the hammer icon does not appear, double-check the file for typos.
</Warning>

## Option B: Claude Code

Claude Code is the terminal-based tool. Use this if you work in a code editor or terminal.

### Project-Level Config (Recommended)

Create a file called `.mcp.json` in your project root directory:

```json
{
  "mcpServers": {
    "ghl": {
      "url": "https://services.leadconnectorhq.com/mcp/",
      "headers": {
        "Authorization": "Bearer YOUR_PIT_TOKEN_HERE",
        "locationId": "YOUR_LOCATION_ID_HERE"
      }
    }
  }
}
```

Replace `YOUR_PIT_TOKEN_HERE` with your actual PIT token. Replace `YOUR_LOCATION_ID_HERE` with your Location ID.

**What you should see:** Claude Code picks up the MCP server the next time it starts in that directory.

### Global Config (All Projects)

If you want Claude Code to have GHL access in every project, save the same config to your global location:

```powershell
notepad "$env:USERPROFILE\.claude\.mcp.json"
```

Paste the same JSON block and save.

## Option C: Self-Hosted Server (Advanced)

<Note>
  Most users do not need this option. The direct endpoint above works without hosting anything. This path is for users who want to run their own MCP server locally.
</Note>

If you want to run the community MCP server locally, follow these steps in PowerShell:

```powershell
git clone https://github.com/mastanley13/GoHighLevel-MCP.git
cd GoHighLevel-MCP
npm install
```

Create the environment file:

```powershell
Copy-Item .env.example .env
notepad .env
```

In the `.env` file, fill in these values:

```
GHL_API_KEY=your_pit_token_here
GHL_BASE_URL=https://services.leadconnectorhq.com
GHL_LOCATION_ID=your_location_id_here
```

Save the file, then build and start the server:

```powershell
npm run build
npm start
```

Test that the server is running:

```powershell
Invoke-RestMethod -Uri "http://localhost:8000/health"
```

**What you should see:** A response confirming the server is healthy.

Then update your Claude config to point to the local server instead of the official endpoint:

```json
{
  "mcpServers": {
    "ghl": {
      "url": "http://localhost:8000",
      "headers": {
        "Authorization": "Bearer YOUR_PIT_TOKEN_HERE",
        "locationId": "YOUR_LOCATION_ID_HERE"
      }
    }
  }
}
```

## Verifying the Connection

No matter which option you chose, verify by opening Claude and checking for tool access. In Claude Desktop, click the hammer icon and look for GHL-related tools in the list.

**What you should see:** A list of available GHL tools like `search_contacts`, `get_contact`, `create_opportunity`, and similar.
