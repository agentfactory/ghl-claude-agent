# Step 4: Run Your First Test

Your connection is configured. Now let us make sure it actually works.

## The Test Prompt

Open Claude (Desktop or Code) and type this:

```
List all contacts in my GoHighLevel account.
```

Press Enter and wait a few seconds.

**What you should see:** Claude returns a list of contacts from your GHL account, including names and basic details. If your account has many contacts, Claude may show the first batch with an option to see more.

[SCREENSHOT: Claude returning a list of GHL contacts]

## What Success Looks Like

If the connection is working, Claude will:

- Reference specific contact names from your account
- Show real data, not example or placeholder information
- Respond within a few seconds

This confirms that Claude is reading directly from your live GHL account.

## Try a Second Test

Now try something that writes data. Pick a test contact (or create one first) and try:

```
Find the contact named [Test Contact Name] and add the tag "claude-test" to their profile.
```

Replace `[Test Contact Name]` with an actual contact in your account.

**What you should see:** Claude confirms the tag was added. You can verify by opening that contact in GHL and checking their tags.

## Troubleshooting

### "I don't see any GHL tools"

Claude does not recognize the MCP connection.

- **Check your config file.** Open it again and verify the JSON is valid. Look for missing commas, brackets, or typos.
- **Restart Claude.** Close it completely (check the system tray) and reopen.
- **Verify the file location.** For Claude Desktop, the file must be at `%APPDATA%\Claude\claude_desktop_config.json`.

### "Authorization failed" or "Invalid token"

Your PIT token is not working.

- **Check for extra spaces.** Make sure there are no spaces before or after your token in the config file.
- **Regenerate the token.** Go back to GHL > Settings > Integrations > Private Integrations and create a new one if needed.
- **Confirm the Bearer prefix.** The Authorization header must say `Bearer ` (with a space) followed by your token.

### "No contacts found" (but you know you have contacts)

The scope or location may be off.

- **Check your scopes.** Make sure `contacts.readonly` is enabled on your Private Integration.
- **Verify the Location ID.** If you are using the self-hosted option, confirm your `.env` file has the correct `GHL_LOCATION_ID`.
- **Try a different query.** Ask Claude to "search for contacts with the tag 'customer'" or a tag you know exists.

### Claude responds but seems slow

This is usually normal for the first request.

- The initial connection takes a moment to establish.
- Requests involving many contacts or records take longer.
- GHL has rate limits that may slow down bulk operations.

Subsequent requests should be faster.
